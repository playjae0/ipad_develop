[프로젝트 파일별 함수/기능 설명]

1) 루트
- app.py
  - _initialize_auth_state(): 인증 관련 session_state 기본값 초기화.
  - _render_unauthenticated_router(): 로그인/회원가입 페이지 라우팅.
  - _render_authenticated_router(): 업로드/라벨링/관리자 페이지 라우팅과 로그아웃 처리.
  - main(): 세션/DB 초기화 후 인증 상태에 따라 앱 시작.
- config.py
  - 함수는 없음. 업로드/ATIS/저장/인증 경로 및 상수 설정 제공.

2) src/auth
- src/auth/password_utils.py
  - hash_password(password): PBKDF2 기반 비밀번호 해시 생성.
  - verify_password(password, password_hash): 입력 비밀번호와 저장 해시 검증.
- src/auth/user_store.py
  - get_connection(db_path): SQLite 연결 생성(row_factory 포함).
  - create_user(...): 사용자 생성.
  - find_user_by_employee_id(...): 사번으로 사용자 조회.
  - find_user_by_email(...): 이메일로 사용자 조회.
  - update_user_status(...): 승인 상태/승인일 갱신.
  - deactivate_user(...): 사용자 비활성화.
  - list_users(...): 사용자 목록 조회.
  - record_login_history(...): 로그인 이력 저장.
  - reset_user_password(...): 비밀번호 해시 재설정.
  - list_successful_login_history(...): 성공 로그인 이력 조회.
  - log_dataset_access(...): 데이터셋 접근 로그 저장.
  - list_dataset_access_history(...): 데이터셋 접근 이력 조회.
- src/auth/auth_manager.py
  - verify_login(...): 로그인 자격 및 계정 상태 검증 + 성공 이력 기록.
- src/auth/db_init.py
  - initialize_auth_db(...): auth 관련 테이블 생성 및 초기 관리자 계정 보장.

3) src/pages
- src/pages/login_page.py
  - render_login_page(db_path): 로그인 폼 렌더링 및 인증 성공 시 세션 반영.
- src/pages/signup_page.py
  - render_signup_page(db_path): 회원가입 폼 처리 및 pending 계정 생성.
- src/pages/upload_page.py
  - render_upload_page(): 업로드 방식 선택, 파일/폴더 입력, 검증 및 라벨링 페이지 진입 처리.
  - _render_uploaded_files_save_section(uploaded_files): 드래그 업로드 원본 저장 보조 UI.
  - _render_folder_selector(): line/period 2단계 폴더 선택 후 파일 수집.
  - _render_validation_result(uploaded_files): 확장자/개수/파싱 검증, master_df 생성, 세션 저장.
  - _render_missing_counts(master_df): 위치별 누락 수량 표시.
  - _render_csv_status_overview_table(): 결과 폴더 기준 최신 CSV 저장 현황 테이블 표시.
- src/pages/labeling_page.py
  - render_labeling_page(): 라벨링 메인 화면 렌더링.
  - _resolve_and_store_image_loading_strategy(...): 관리자 설정/이미지 수 기준 로딩 전략 결정 및 상태 반영.
  - _build_runtime_image_map(...): eager/lazy_cache 전략에 따라 렌더용 이미지 맵 생성.
  - _load_visible_images(...), _preload_neighbor_images(...), _evict_old_cached_images(...): lazy rolling cache 보조.
  - _render_sidebar_source_info(): 선택된 폴더 정보 표시.
  - _render_sidebar_previous_csv_loader(): 이전 CSV 값 불러오기(2단계 확인) UI.
  - _render_save_section(df, image_map): CSV/이미지 저장 UI.
  - _resolve_csv_output_dir_and_filename(employee_id): CSV 저장 대상 폴더/파일명 결정.
  - _render_navigation_buttons(current_index, total_count): 이전/다음 셀 이동 UI.
  - _safe_index(index, length): 인덱스 범위 보정.
- src/pages/admin_page.py
  - render_admin_page(): 사용자/이력/락 모니터링 + 이미지 로딩 전략 설정 화면.
  - _render_image_loading_strategy_section(): image_loading_mode/eager_threshold/preload count 설정 UI.
  - _load_image_loading_settings(), _save_image_loading_settings(): SQLite app_settings 저장/조회.

4) src/ui
- src/ui/defect_controls.py
  - render_defect_selector(...): 상위 단일선택 + 하위 라벨 버튼 렌더링 및 선택값 반환.
- src/ui/image_grid.py
  - render_image_grid(...): 현재 cell의 포지션별 이미지/결함 UI 렌더링.
  - _render_single_position(...): 단일 포지션 카드 렌더링.
  - _get_atis_value(...): ATIS 상위값 조회(결측/비정상값은 OK 정규화).
  - _apply_atis_override(...): 상위값 변경 시 ATIS `Original/New` 반영.
  - _to_image_source(image_ref): 표시 가능한 이미지 소스로 변환.
- src/ui/sidebar_list.py
  - render_sidebar_cell_list(df, current_index): cell 목록/선택 UI.
  - _build_cell_summary(df, index): sidebar 표시용 요약 문자열 생성.
- src/ui/status_panel.py
  - render_status_panel(df, current_index): 진행 상태/메트릭 패널 렌더링.

5) src/core 성격 모듈
- src/file_parser.py
  - FileParseResult: 파일명 파싱 결과 데이터 모델.
  - parse_filename(file_reference): 파일명에서 cell_id/position 파싱.
  - _extract_filename(file_reference): 참조 객체에서 파일명 추출.
- src/validation.py
  - validate_file_extensions(file_references): 허용 확장자 검증.
  - validate_file_count(file_references): 업로드 개수 검증.
  - parse_files_with_results(file_references): 파일별 파싱 결과 생성.
  - extract_parse_failures(parsed_pairs): 파싱 실패 목록 생성.
  - _extract_filename(file_reference): 파일명 추출 보조.
- src/image_registry.py
  - register_image(image_map, parse_result, file_reference): 파싱 결과를 image_map에 반영.
  - build_image_map(parsed_pairs): 전체 image_map 구축.
  - load_image_bytes(file_reference): 이미지 참조에서 실제 bytes 지연 로딩.
- src/dataframe_builder.py
  - build_master_dataframe(image_map): image_map 기반 master dataframe 생성.
- src/atis_loader.py
  - merge_atis_to_master(master_df): ATIS 파일을 master_df에 병합(가능 시).
  - _normalize_cell_id(value): cell_id 비교용 정규화.
- src/state_manager.py
  - initialize_session_state(): 세션 기본값 초기화.
  - set_master_dataframe/get_master_dataframe(): master_df 저장/조회.
  - set_image_map/get_image_map(): image_map 저장/조회.
  - set_current_cell_index/get_current_cell_index(): 현재 인덱스 저장/조회.
  - set_upload_completed/is_upload_completed(): 업로드 완료 상태 저장/조회.
  - touch_label_sync_token/get_label_sync_token(): 라벨 동기화 토큰 관리.
  - set_image_loading_settings/get_image_loading_settings(): 이미지 로딩 설정 상태 저장/조회.
  - set_resolved_loading_strategy/get_resolved_loading_strategy(): 최종 로딩 전략 저장/조회.
  - set_sidebar_cell_index/get_sidebar_cell_index(): 사이드바 선택 인덱스 동기화.
- src/save_manager.py
  - build_csv_export_payload(...): 기본 CSV 다운로드 파일명/바이트 생성.
  - build_csv_filename(...): CSV 파일명 생성.
  - save_csv_to_path(...): CSV 파일 저장.
  - parse_version_from_filename(...): 파일명에서 버전 파싱.
  - find_latest_csv_version(...): 폴더 내 최신 버전 조회.
  - build_next_version(...): 다음 버전 계산.
  - build_next_version_filename(...): 버전 포함 CSV 파일명 구성.
  - ensure_result_folder_from_selected_subpath(...): 결과 폴더 보장 생성.
  - export_csv_without_filling_ok(...): 값 변경 없이 CSV 저장.
  - find_latest_csv_file(...): 폴더 내 최신 CSV 파일 경로 조회.
  - extract_employee_and_version_from_filename(...): 파일명에서 employee/version 추출.
  - load_previous_defect_values(...): 이전 CSV에서 defect 컬럼 로딩.
  - apply_loaded_defect_values(...): 현재 df에 defect 컬럼만 덮어쓰기.
  - save_defect_images(...): 결함 이미지 내보내기 저장.
  - _infer_extension(...), _read_image_bytes(...): 이미지 저장 보조.

6) utils
- utils/io_utils.py
  - dataframe_to_csv_bytes(df, encoding): DataFrame -> CSV bytes.
  - get_timestamp(fmt): 현재 시각 문자열 생성.
- utils/naming_utils.py
  - sanitize_token(raw, fallback): 파일/폴더명 안전 문자열 변환.
  - build_labeled_image_filename(...): 이미지 저장 파일명 구성.
  - infer_extension_from_path(file_name_or_path): 확장자 추론.
- utils/path_utils.py
  - ensure_directory(path): 폴더 생성/보장.
  - build_session_save_path(save_root, session_name): 세션 저장 경로 생성.
  - list_subdirectories(root_path): 1-depth 하위 폴더 목록.
  - list_subdirectories_relative(root_path): 재귀 상대 하위 폴더 목록.
  - collect_files_with_extensions(directory, extensions): 확장자 필터 파일 수집.

(참고)
- __init__.py, constants.py는 주로 모듈 export/상수 정의 파일로, 독립 함수는 거의 없음.
