[프로젝트 전체 기능 설명]

1. 프로젝트 목적
- Streamlit 기반의 결함 라벨링 플랫폼입니다.
- 이미지 업로드 -> 파일명 파싱 -> master dataframe 생성 -> 라벨링 -> CSV/이미지 저장 흐름을 제공합니다.
- 사내 계정 기반 인증(로그인/회원가입/관리자)을 통해 접근을 제어합니다.

2. 사용자 흐름
(1) 인증
- 로그인/회원가입 페이지를 통해 사용자 인증을 수행합니다.
- 승인/활성 상태를 검사하여 접근 가능 여부를 결정합니다.
- 관리자 계정은 초기 DB 세팅 시 자동 보장됩니다.

(2) 업로드
- 드래그 업로드 또는 폴더 선택 업로드를 지원합니다.
- 폴더 선택 시 `line/period` 2단계 경로를 선택합니다.
- 파일 개수/확장자/파일명 파싱을 검증하고, 유효한 데이터로 image_map 및 master dataframe을 생성합니다.
- 업로드 후 라벨링 페이지로 이동합니다.

(3) 라벨링
- cell_id 단위로 이미지를 탐색하며 포지션별 결함값을 지정합니다.
- 상위 결함(OK/Damage/Crack/Scrap) 1개를 선택하고, 해당 상위에 매핑된 하위 라벨을 선택합니다.
- Defect_* 컬럼에는 하위 라벨만 저장되며, 상위만 선택한 상태는 저장값이 없습니다.
- ATIS 값이 변경되면 ATIS 컬럼에 `Original/New` 형식으로 override를 기록합니다.
- 이미지 로딩 전략(eager/lazy_cache)은 관리자 설정값에 따라 자동/수동 결정됩니다.
- 사이드바 목록/상태 패널로 진행 상황을 확인합니다.
- 필요 시 같은 폴더의 최신 CSV를 불러와 defect 컬럼만 덮어쓸 수 있습니다(2단계 확인).

(4) 저장/내보내기
- CSV 저장: 선택된 업로드 소스/폴더를 기준으로 경로와 파일명을 구성하고 저장합니다.
- 버전 관리: 기존 파일 버전을 스캔해 다음 버전을 자동 계산합니다.
- 이미지 저장: 포지션/결함 기준의 폴더 구조로 결함 이미지를 저장합니다.

3. 데이터 구조 핵심
- image_map: `cell_id -> position -> image reference` 구조.
- master dataframe:
  - 기본 컬럼: cell_id + 4개 포지션 컬럼
  - defect 컬럼: Defect_CA(TOP), Defect_CA(BOT), Defect_AN(TOP), Defect_AN(BOT)
- 라벨링 시 defect 컬럼 값이 갱신되며, CSV/이미지 저장에 사용됩니다.

4. 인증/권한/이력
- SQLite 기반 사용자/로그인 이력/데이터셋 접근 이력을 관리합니다.
- 로그인 성공 시 이력을 남기고,
- 업로드 경로 선택 후 진행 시 데이터셋 접근 로그를 남깁니다.
- 관리자 페이지에서 주요 이력을 조회할 수 있습니다.
- 관리자 페이지에서 이미지 로딩 전략 설정(image_loading_mode, eager_threshold, preload count)을 변경할 수 있습니다.

5. 경로/파일 정책(요약)
- 이미지 루트: `data/images`
- CSV 결과 루트: `output/result`
- 이미지 export 루트: 설정값 기반(`outputs` 계열)
- 폴더 선택 업로드 시 이미지 경로와 CSV 경로는 동일 상대경로를 사용합니다.
- CSV 파일은 버전(`verX.Y`)을 포함해 저장됩니다.

6. 모듈 구성
- 페이지/UI: `src/pages`, `src/ui`
- 도메인 로직: `src/file_parser`, `src/validation`, `src/image_registry`, `src/dataframe_builder`, `src/save_manager`
- 인증: `src/auth/*`
- 공통 상태/상수: `src/state_manager`, `src/constants`
- 유틸: `utils/*`
- 앱 엔트리/설정: `app.py`, `config.py`

7. 운영 시 유의사항
- 파일명 규칙을 벗어난 이미지는 파싱 실패로 제외될 수 있습니다.
- 라벨링 데이터는 세션 상태를 중심으로 동작하므로 브라우저 세션 변화에 유의해야 합니다.
- CSV 불러오기 기능은 defect 컬럼만 덮어쓰며, 다른 컬럼/구조는 유지됩니다.
- 라벨링 페이지의 셀 선택은 사이드바 선택/이전·다음 버튼이 동일 상태로 동기화됩니다.
